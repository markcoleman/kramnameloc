Two weeks ago an associate requested to shadow myself and the development team to learn more about what we do as they are interested in development.  I unfortunately only had two hours with them to dump tons of information into their brain which most likely resulted in overload.  To help combat this I provided a list of random topics to help them learn and learn about in the future.  I present you my list of random things that is no where complete but a starting point to learn about development.

##General
 - [Stackoverflow](http://www.stackoverflow.com): The best QA site on development topics [(my profile)](http://stackoverflow.com/users/181776/mark-coleman)
 - [Codecademy](http://www.codecademy.com): Learn to code in your browser
 - [Pluralsight](http://www.pluralsight.com): Online training for all things development and IT, free 90 day trial for students
 - Learn [git](https://try.github.io/levels/1/challenges/1): When I went to college they never taught source control, it would have been so helpful early on.

##HTML Stuff
 - [JsFiddle](http://jsfiddle.net): Zero friction html/js/css site when sometimes you want to try something without launching an IDE or a web server. [my profile](http://jsfiddle.net/user/markcoleman/fiddles/)
 - [Plnkr ](http://plnkr.co): Zero friction html/js/css site, also very popular a bit more robust.
 - [Firebug](http://getfirebug.com): My defacto web debugger
 - [Bower](http://bower.io): Web package management, why hunt for packages when you can install than via command line.

##.NET
 - [C# virtual academy](http://www.microsoftvirtualacademy.com/training-courses/c-fundamentals-for-absolute-beginners): Learn C# from Microsoft
 - [dotnetfiddle.net](https://dotnetfiddle.net): Zero friction C# development in your browser. 
 - [nuget.org](http://www.nuget.org): Package management for .net projects.

##JavsScript
 - [Jquery](https://www.jquery.com): If you do web development it is great to learn as there are a plethora of plugins.
 - [AngularJs](https://angularjs.org): One of the many web frameworks, currently my favorite.
 - [NodeJs](https://nodejs.org): Love JavaScript? how about javascript outside of the browser.

##IDE/Tools
 - [Visual Studio Community Edition](https://www.visualstudio.com/en-us/products/visual-studio-community-vs.aspx): Now free so no reason not to use it especially if you do C#
 - [ReSharper](https://www.jetbrains.com/resharper/): Alt-Enter all the things! Helps in refactoring, code style, templates, reveals potential bugs, and so much more.
 - [WebStorm](https://www.jetbrains.com/webstorm/): My favoriate html/JavaScript IDE
 - [Sublimetext](http://www.sublimetext.com): Don't use it but still very popular.

##Books
 - [C# in depth](http://www.manning.com/skeet3/): Learn C# from the [master](http://stackoverflow.com/users/22656/jon-skeet)
 - [Clean Code](http://www.amazon.com/gp/product/0132350882/ref=pd_lpo_sbs_dp_ss_1?pf_rd_p=1944687742&pf_rd_s=lpo-top-stripe-1&pf_rd_t=201&pf_rd_i=0137081073&pf_rd_m=ATVPDKIKX0DER&pf_rd_r=1ZB0FFBKY429VSX933G0): Learn about good and bad code
 - [Code Complete](http://www.amazon.com/Code-Complete-Practical-Handbook-Construction/dp/0735619670/ref=pd_sim_b_2?ie=UTF8&refRID=1TJE4WY815WTNFZ0RG0R): The definitive guide to what it means to produce clean high quality code.
